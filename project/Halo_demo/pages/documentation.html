Halo - 文档

1. 这是什么（What is It？）

Halo是为开发网站和Web App而创建的移动模板。 由于流畅的布局，它适合任何移动设备的类型。 此外，使用预构建的类，设计新页面或修改新页面将更容易已创建。
该模板使用Google Framework Material Design Light（MDL）来强制执行Material Design指南。结果页面转换会影响用户与应用程序类似的体验。

2. 文件夹结构（Folders Structure）

您可以在下面找到文件夹结构的简短摘要：

  • 在名为“app”的文件夹中列出了模板文件，分为html模板和ajax导航（适用于更多专家用户）。
  • 在名为“css”的文件夹中列出了样式表：
    • keyframes.css包含动画的规则。不要直接修改这些文件，否则新的更新可能会覆盖您的工作；
    • materialize.min.css包含框架的规则。不要直接修改这些文件，否则新的更新可能会覆盖您的工作；
    • swiper.css包含滑块的规则。不要直接修改这些文件，否则新的更新可能会覆盖您的工作；
    • swipebox.min.css包含灯箱的规则。不要直接修改这些文件，否则新的更新可能会覆盖您的工作；
    • style.css包含自定义规则。在这里，您可以添加自己的规则，以便修改或覆盖以前的文件。
  • 在名为“js”的文件夹中列出了javascript规则：
    • jquery-2.1.0.min.js是一个框架javascript。不要直接修改这些文件;
    • jquery.swipebox.min.js包含灯箱的javascript规则。不要直接修改这些文件;
    • jquery.smoothState.min.js包含路由的javascript规则。不要直接修改这些文件;
    • materialize.min.js包含框架的javascript规则。不要直接修改这些文件;
    • swiper.min.js包含滑块的javascript规则。不要直接修改这些文件;
    • jquery.mixitup.min.js包含图像过滤器的javascript规则。不要直接修改这些文件;
    • masonry.min.js包含砌体的网格javascript规则。不要直接修改这些文件;
    • chart.min.js包含图表的javascript规则。不要直接修改这些文件;
    • function.js包含自定义规则。在这里，您可以添加自己的规则以修改模板或覆盖以前的文件。
  • 在名为“img”的文件夹中列出模板使用的图像。

3. 布局结构（Layout Structure）

文件.html分为两部分

  • “header”标签包含用于打开侧边栏的按钮;
  • “main”包含内容和页面的主要结构。这是从页面到页面的基本变化;
  • 页面以侧边栏，工具栏，内容和页脚开头。

5. 动画（Animations）

Halo使用SmoothState进行页面路由/转换。 SmoothState.js是一个逐步增强的jQuery插件页面加载，以控制页面转换。 如果用户的浏览器没有所需的功能，smoothState.js淡入背景并且永远不会运行。 smoothState.js提供了可用于的钩子编排元素在导航过程中如何进入和退出页面。 它使用动画运行的时间通过AJAX获取内容以注入页面。 SmoothState.js没有规定页面上的内容应该如何动画。 它支持CSS动画以及JS动画库。
以下转换是预构建的，因此您只需添加该类：

  • 淡入
  • 向上淡入
  • 向下淡出
  • 向右淡入
  • 原位置出现

You can add a delay too:

  • delay-1
  • delay-2
  • delay-3
  • delay-4
  • delay-5
  • delay-6

6. 自定义Javascript（Custom Javascript）

工作路由使用ajax，因此您需要在页面加载后调用每个javascript代码。 这样做有在functions.js中的一个函数叫做“initiate_plugins”，只是在里面重写你的函数。
如果您不是专家，我们建议您使用html文件夹中的模板版本。
可以使用以下条件：

一个jQuery选择器，指定应绑定smoothState元素中的哪些锚点。

			
				$('#main').smoothState({ anchors: 'a' });

A jQuery selector specifying which forms within the smoothState element should be bound.

			
				$('#main').smoothState({ forms: 'form' });

控制是否表单提交响应保留在缓存中。 如果设置为true，则smoothState将将表单响应存储在缓存中。 除非您了解缓存形式的结果，否则应将其设置为false很好地影响您网站的行为。

			
				$('#main').smoothState({ allowFormCaching: false });

一个jQuery选择器，指定应忽略smoothState元素中的哪些元素。 这包括两者形式和锚元素。

			
				$('#main').smoothState({ blacklist: '.no-smoothState' });

要缓存的页数。smoothState.js可以在内存中缓存页面，避免用户不得不请求页面不止一次。 缓存页面将立即加载。

			
				$('#main').smoothState({ cacheLength: 0 });
